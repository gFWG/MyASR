JSON Export with Highlights Test
=================================

When include_highlights=True (default) and format="json", the output contains:
- All original sentence record fields
- "vocab_highlights": array of vocab highlight objects
- "grammar_highlights": array of grammar highlight objects

Example output structure:
[
  {
    "id": 1,
    "japanese_text": "日本語のテキスト",
    "chinese_translation": "中文翻译",
    "explanation": "説明",
    "source_context": "context",
    "created_at": "2024-01-15T10:30:00",
    "vocab_highlights": [
      {
        "surface": "日本語",
        "lemma": "日本語",
        "jlpt_level": 3,
        "pos": "名詞"
      },
      {
        "surface": "テキスト",
        "lemma": "テキスト",
        "jlpt_level": null,
        "pos": "名詞"
      }
    ],
    "grammar_highlights": [
      {
        "pattern": "の",
        "jlpt_level": 5,
        "description": "Possessive particle"
      }
    ]
  }
]

SQL queries used (parameterized with ?):
1. SELECT surface, lemma, jlpt_level, pos FROM highlight_vocab WHERE sentence_id = ?
2. SELECT pattern, jlpt_level, description FROM highlight_grammar WHERE sentence_id = ?

No f-strings in SQL - all queries use parameterized placeholders.
