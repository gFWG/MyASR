CSV Export with Highlights Test
================================

When include_highlights=True (default) and format="csv", the output contains:
- All original columns: id, japanese_text, chinese_translation, explanation, source_context, created_at
- 4 additional columns: vocab_count, grammar_count, vocab_lemmas, grammar_rules

CSV header with highlights:
id,japanese_text,chinese_translation,explanation,source_context,created_at,vocab_count,grammar_count,vocab_lemmas,grammar_rules

Example row:
1,日本語のテキスト，中文翻译，説明，context,2024-01-15T10:30:00,2,1,日本語;テキスト，の

Column definitions:
- vocab_count: integer count of vocab highlights for this sentence
- grammar_count: integer count of grammar highlights for this sentence
- vocab_lemmas: semicolon-joined lemma strings (e.g., "食べる;走る")
- grammar_rules: semicolon-joined pattern strings (e.g., "の;を")

SQL queries used (parameterized with ?):
1. SELECT lemma FROM highlight_vocab WHERE sentence_id = ?
2. SELECT pattern FROM highlight_grammar WHERE sentence_id = ?

No f-strings in SQL - all queries use parameterized placeholders.
