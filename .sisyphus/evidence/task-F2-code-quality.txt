FIX APPLIED: Changed @patch("src.pipeline.AudioCapture") → @patch("src.pipeline.create_audio_capture")
in all 12 test functions in tests/test_pipeline.py.

Root cause: pipeline.py imports the factory function create_audio_capture from
src.audio.capture, not the AudioCapture class directly. Patching AudioCapture
in the pipeline module's namespace failed because the name doesn't exist there.
Patching create_audio_capture intercepts the factory call; mock_audio_cls.return_value
becomes the mock audio capture instance, matching how tests already used it.

ADDITIONAL BUG FIXED in src/pipeline.py:
  is_beyond_level comparison was inverted. JLPT levels are N1(=1,hardest)..N5(=5,easiest).
  A user at level 3 finds N1(=1) vocab "beyond their level" because it's harder.
  "Beyond level" means word is harder = numerically LOWER jlpt_level.
  Changed: is_beyond_level=vh.jlpt_level > self._user_level (WRONG: 1>3=False)
       to: is_beyond_level=vh.jlpt_level < self._user_level (CORRECT: 1<3=True)
  Same fix applied to grammar_recs. Confirmed by test_to_db_records_converts_correctly.
