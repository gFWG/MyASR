FILE: src/pipeline/llm_worker.py (139 lines)
  Status: ISSUES FOUND
  Issues:
    [MEDIUM] llm_worker.py:45-46 — `llm_client: Any` and `db_repo: Any` —
      both are typed as Any with no comment explaining why. The llm_client
      should be typed as `AsyncOllamaClient` (it's imported in orchestrator.py
      and passed directly). The db_repo should be typed as
      `LearningRepository | None` (it can be None when no DB path is set, per
      orchestrator usage). Using `Any` defeats mypy's ability to catch interface
      mismatches at the worker level. AGENTS.md: "Avoid Any. If truly needed,
      comment why."
      Locations: llm_worker.py:45, llm_worker.py:46

    [MINOR] llm_worker.py:115 — `except Exception as exc:  # noqa: BLE001`
      — broad exception suppressed with noqa. The noqa is documented with
      the ruff code (BLE001 = blind exception) but there is no human-readable
      comment explaining WHY we catch all exceptions here (to prevent DB errors
      from crashing the LLM thread). Add a comment: `# DB errors must not
      propagate into the LLM thread`.

    [MINOR] llm_worker.py:109-120 — `self._db_repo.update_translation(...)` is
      called unconditionally even when `self._db_repo` could be None (as seen
      in orchestrator where db_path defaults to ":memory:" and a real repo is
      always created — but if LlmWorker is tested or used standalone, a None
      db_repo would crash at line 110 with AttributeError, not a clean error).

FILE: src/pipeline/orchestrator.py (205 lines)
  Status: ISSUES FOUND
  Issues:
    [MEDIUM] orchestrator.py:160, 165, 170 — Properties `asr_ready`,
      `translation_ready`, `error_occurred` return `Any` / `list[Any]`. These
      should return proper Qt Signal types. The lack of typing here means callers
      get no IDE/mypy support for `.connect()`. The correct type would be
      `SignalInstance` from PySide6 stubs, or at minimum a comment explaining the
      Any. AGENTS.md: "Avoid Any. If truly needed, comment why."
      Locations: orchestrator.py:160, 165, 170

    [MEDIUM] orchestrator.py:142-143 — `on_asr_ready: Any` and
      `on_translation_ready: Any` in `connect_signals` — these should be typed
      as `Callable[[ASRResult], None]` and `Callable[[TranslationResult], None]`
      respectively to give callers type safety.

    [MINOR] orchestrator.py:201 — `import dataclasses` is a deferred import
      inside `_build_app_config()`. Deferred imports are sometimes used to avoid
      circular imports, but dataclasses is stdlib and there is no circular import
      risk here. Should be moved to the module-level imports. This is a style
      violation against the project's import conventions.

    [MINOR] orchestrator.py:56 — `SileroVAD` and `QwenASR` are instantiated
      inside `__init__` (heavy GPU objects). The class docstring says models are
      "injected at construction time" but that's misleading — they are CREATED
      here, not injected. Either fix the docstring or refactor to true DI to
      enable unit testing without GPU.
