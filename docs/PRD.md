# PRD：日语学习悬浮插件（系统音频字幕 + 翻译 + JLPT 高亮 + 考点解析）

## 1. 背景与目标

### 1.1 问题陈述
用户在玩游戏/看电影/刷视频时，会遇到听不清、跟不上、词汇/语法超纲等问题，需要一个不打断体验的“悬浮字幕 + 翻译 + 考点提示”来辅助学习。

### 1.2 目标（Goals）
- G1: 捕获系统播放音频并进行语音切段（VAD），得到较完整的日语句子级文本字幕。
- G2: 对字幕进行预处理（分词、JLPT 词库查询、语法规则匹配、复杂度评分），决定是否触发更详细的考点解析。
- G3: 在桌面透明悬浮窗中展示“日语原文 + 中文翻译 + JLPT 高亮”，并支持鼠标悬停弹出解释与记录学习条目。
- G4: 所有核心能力本地离线运行（ASR 离线推理 + 本地 Ollama LLM），降低网络依赖与延迟。

### 1.3 成功指标（Success metrics）
- M1: 跑通预处理链路（形态分析 + 词汇查询 + 语法匹配 + 复杂度评分）。
- M2: 接入 qwen-ASR 模型，成功得到识别结果。
- M3: 成功调用本地/云端LLM，得到翻译与解析。
- M4: 实现字幕和悬浮 Tooltip 。

### 1.4 非目标（Non-goals）
- NG1: 不做复杂架构/机制。
- NG2: 不做流式 ASR（优先使用 VAD 切段后整句推理来提高复杂场景准确率）。
- NG3: 不追求通用多语言学习，当前聚焦日语字幕学习与 JLPT 体系标注。

***

## 2. 用户与使用场景

### 2.1 目标用户
- 在 Windows 11 上玩日语游戏/看日语视频，希望实时看到字幕、翻译，并能把不懂的词汇/语法点快速沉淀为学习记录的用户。

### 2.2 用户故事（按优先级）
- P0: 作为用户，我想让软件捕获“系统播放音”并自动生成日语字幕，以便我不需要手动打字或截图识别。
- P0: 作为用户，我想在悬浮窗上同时看到日语原文与中文翻译，以便我能一边看内容一边理解。
- P0: 作为用户，我想让超纲词汇/语法点按 JLPT 等级高亮，以便我快速定位学习重点。
- P0: 作为用户，我想鼠标悬停高亮词时弹出解释并自动加入学习记录，以便积累复习素材。
- P1: 作为用户，我想设置自己的 JLPT 水平（N5~N1）并实时生效，以便不同阶段看到不同的“超纲”提示。
- P1: 作为用户，我想导出学习记录为 CSV/JSON，以便做备份或用别的工具复习。
- P1: 作为用户，我想在“学习面板”里按时间/关键词搜索与排序记录，以便快速回顾。
- P2: 作为用户，我想有“复习功能”，按某种算法提示应该巩固的句子并查看解析。

## 3. 范围

### 3.1 功能清单
- P0:
- 捕获系统音频（非麦克风）并用 Silero VAD 切段。

- 学习面板：历史记录表格（排序/搜索）。

- P2:
- 复习功能（算法待定）。

## 4. 体验与页面

### 4.1 页面/路由清单（桌面应用视角）
| 页面/窗口 | 目的 | 关键状态(加载/空/错) |
|---|---|---|
| 悬浮窗（Overlay） | 实时展示日语字幕/翻译与高亮标注  | 加载：模型初始化中；空：无语音/无字幕；错：音频设备不可用/ASR 失败/LLM 超时  |
| Tooltip（悬停弹窗） | 展示词汇/语法点解析并触发写入学习记录  | 空：词库未命中；错：写入失败/解析失败  |
| 设置面板（Settings） | 调整 JLPT 水平、外观、快捷键、模型/资源、记录导出  | 空：未配置模型；错：资源缺失/路径无权限  |
| 学习面板（History/Review） | 查看、搜索、排序历史学习记录与复习入口  | 空：无记录；错：DB 损坏/查询失败  |

### 4.2 核心用户流程
1) 用户启动应用 → 应用捕获系统音频流 → VAD 切段。
2) 句子结束触发 ASR 离线推理 → 输出完整日语句子（避免截断）。
3) 预处理链路执行：分词、JLPT 标注、语法匹配、复杂度评分 → 决策简单/复杂句路径。
4) LLM 单次调用：简单句仅翻译，复杂句翻译 + 考点解析 → 返回结果。
5) 悬浮窗展示原文/译文与高亮 → 用户悬停高亮词弹出 Tooltip → 自动写入 SQLite 学习记录（同句只写一次）。

## 5. 需求明细

### 5.1 音频捕获与切段（VAD）
- 描述：用 `sounddevice` + `pyaudiowpatch` 捕获系统播放音，并用 Silero VAD 区分人声与背景音，确认句子结束后输出完整片段。
- 规则/校验：VAD 参数允许按场景调整（游戏/电影/视频差异）。
- 错误与空状态：未检测到语音时悬浮窗显示空状态；音频设备/权限异常时提示错误。
- 验收标准（Given-When-Then）：
- Given 用户正在播放包含人声的系统音频，When VAD 检测到一句话结束，Then 输出一个完整的音频片段并触发后续 ASR。
- Given 当前只有 BGM/音效无人声，When VAD 处理音频流，Then 不应频繁触发 ASR 推理（避免噪声触发）。

### 5.2 ASR：Qwen3-ASR 离线推理
- 描述：使用 Qwen3-ASR（0.6B）离线模式，VAD 回调触发，输入完整语音片段输出完整日语句子。
- 规则/校验：不使用流式模式，理由是复杂背景音场景下流式更易“幻觉”，整句推理更稳。
- 错误与空状态：ASR 推理失败时可显示“识别失败/请重试”，并保留日志信息供排查（格式待定）。
- 验收标准：
- Given 模型已加载并常驻显存，When 输入任意有效语音片段，Then 应输出一条日语文本结果或明确失败原因。

### 5.3 预处理：分词 + JLPT + 语法匹配 + 复杂度评分
- 描述：在 LLM 前置执行毫秒级预处理，分三个子模块串行，并输出结构化标注。
- 规则/校验：
- 形态分析：`fugashi` + `unidic-lite`，输出表记形、原形 lemma、词性 POS。
- JLPT 词汇：N1~N5 JSON 词表启动时加载为内存 dict，查询 O(1)，输出等级或“未收录”，并判断是否超出用户水平。
- 语法规则：800+ 语法点 CSV 转为 JSON 正则表，分“高置信度直接命中”和“歧义送审表命中后交给 LLM 裁定”。
- 复杂度评分：结合 `jreadability` 的结构复杂度与 JLPT 加权的词汇复杂度，综合决定是否触发考点解析。
- 验收标准：
- Given 输入一条日语句子，When 预处理执行，Then 返回包含分词、JLPT 标注、语法命中结果与复杂度分数的结构化结果。
- Given 预处理模块开启，When 连续处理多条句子，Then 平均耗时应保持在 < 50ms（需要在实现中加入计时与统计）。

### 5.4 LLM：翻译与考点解析（单次调用）
- 描述：使用本地 Ollama server（localhost:11434）调用 qwen3.5:4b，翻译与考点解析合并单次调用，简单句仅翻译，复杂句翻译 + 考点解析。
- 规则/校验：复杂句的判定来自预处理层综合决策（词汇/语法/复杂度）。
- 错误与空状态：LLM 超时/失败时 UI 要显示“仅字幕/无解析”或降级提示（降级策略细节待补）。
- 验收标准：
- Given 句子被判定为简单句，When 调用 LLM，Then 返回中文翻译且不返回冗长考点解析。
- Given 句子被判定为复杂句，When 调用 LLM，Then 返回中文翻译并包含考点解析（并确保是一次调用产生的结果）。

### 5.5 UI：透明悬浮窗 + Tooltip + 高亮
- 描述：使用 PySide6 做透明悬浮窗，显示日语原文/中文翻译，并对超纲词汇/语法点按 JLPT 等级着色高亮。
- 规则/校验：
- 高亮颜色：例如 N2单词=浅黄；N2语法=深黄；N1单词=浅红；N1语法=深红...(语法优先级高于词汇，即语法覆盖词汇)
- 显示模式：支持“第一行日语第二行中文”或“同一行切换显示”的模式。
- Tooltip：鼠标悬停高亮词时弹出等级 + 简要解释，并自动加入学习记录（同一个高亮在每个句子只能触发一次）。
- 验收标准：
- Given 系统正在识别并产生字幕，When UI 收到新句子结果，Then 悬浮窗应更新显示原文/译文且高亮正确。
- Given 用户悬停某个高亮词，When Tooltip 弹出，Then 展示 JLPT 等级与简要解释，并写入学习记录且同句不重复写入。

### 5.6 设置面板
- 描述：提供设置面板调整 JLPT 水平、外观（字体/颜色/位置）、快捷键、模型接口与参数、资源管理（词库/规则表/模型文件路径）、记录导出与清理等。
- 验收标准：
- Given 用户打开设置面板，When 修改并保存，Then 实时更新。

### 5.7 学习记录
- 描述：使用 SQLite 存储学习记录，包含原文、翻译、讲解、时间戳等字段，并支持导出 CSV/JSON、按时间段删除历史记录。
- 验收标准：
- Given 用户触发 Tooltip 且写入规则满足，When 写入数据库，Then 学习面板可立即检索到新记录。
- Given 用户选择导出，When 导出执行，Then 能生成 CSV/JSON 文件并包含记录字段（字段定义需在实现时固化）。

### 5.8 学习面板
- 描述：展示历史学习记录的表格，支持按时间/关键词搜索与排序，并提供复习入口（复习功能待定）。
- 验收标准：
- Given 用户打开学习面板，When 数据库有记录，Then 应展示记录列表；When 用户搜索/排序，Then 应正确过滤/排序记录。

## 6. 数据与权限

### 6.1 核心实体（字段级）
- SentenceRecord（学习记录）
- id（自增或 UUID，待定）  
- japanese_text（日语原文）
- chinese_translation（中文翻译）
- explanation（考点解析/讲解，可能为空）
- highlights（高亮标注：词汇/JLPT/语法命中，结构待定）
- created_at（时间戳）
- source_context（可选：来源场景如游戏/视频，待定）

- LexiconEntry（词库条目，来自本地 JSON，存储方式待定）
- surface/lemma/pos（来自分词结果）
- jlpt_level（N5~N1 或未收录）

- GrammarRule（语法规则，来自 JSON 正则表）
- rule_id  
- pattern_regex  
- confidence_type（高置信/歧义送审）
- description（简述，待定）

### 6.2 权限
当前为本地单用户桌面应用，默认当前用户有所有权限，不引入额外内容。

## 8. 依赖、风险与未决问题

- 依赖：
- `pyaudiowpatch` 对“系统音频捕获”的兼容性与权限配置。
- 本地资源文件：JLPT 词表 JSON、语法点 CSV→JSON 正则表、ASR 模型文件等需要资源管理模块支撑。
- 本地 Ollama server（localhost:11434）可用性与模型管理（qwen3.5:4b）。

- 风险：
- 音频场景差异导致 VAD 参数难以“一套通吃”，需要提供可调参数或场景预设。
- 语法规则正则命中存在歧义，40% 规则需要 LLM 裁定，可能带来额外延迟或不稳定性。

- 需要确认的问题：
- 单词表/语法表的获取与更新机制
- “复杂句”触发阈值具体怎么定（JLPT 超纲比例/语法命中/可读性分数权重）。
- 学习记录字段的最终结构（尤其 highlights 与 explanation 的 schema）。
- 复习算法采用什么策略
